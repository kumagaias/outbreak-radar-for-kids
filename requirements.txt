子ども感染症レーダー / Outbreak Radar for Kids
ユーザーストーリー v1.0
2026年3月

1. ユーザー像（ペルソナ）
佐藤さん（32歳・葛飾区在住）

3歳の娘を保育園に通わせているワーキングマザー
毎朝バタバタしていて、感染症情報を調べる時間がない
保育園からお知らせが来るたびに「うちの子大丈夫？」と不安になる
厚労省の週報は難しくて読む気になれない


2. ユーザーが抱える状況
場面気持ち欲しいもの許容時間保育園からお知らせが来た不安・焦り「うちの子、大丈夫？」への即答3分以内毎朝登園前のルーティン落ち着いている異常がないかサッと確認30秒以内

3. ユーザーストーリー
Story 1｜はじめて使う
【場面】
アプリを初めて開いた

【ユーザーの行動】
1. アプリを開く
2. 「お子さんの情報を教えてください」画面が表示される
3. 子供の年齢をざっくり選ぶ（0〜1歳 / 2〜3歳 / 4〜6歳 / 7歳以上）
4. お住まいの区・市をドロップダウンで選ぶ
5. 「はじめる」をタップ
6. トップ画面へ遷移

【完了条件】
- プロフィールがlocalStorageに保存される（サーバー送信なし）
- 次回起動時は自動でプロフィールが反映される
- 30秒以内に完了できる

Story 2｜毎朝のルーティンチェック
【場面】
朝7時、子供の身支度をしながらスマホを開く

【ユーザーの行動】
1. アプリを開く
2. トップ画面に「今日の○○ちゃん」の診断結果が即座に表示される
3. ✅ 平常 → そのまま閉じる（10秒で完了）
   ⚠️ 注意 → 「詳しく見る」をタップ → Nova の診断詳細を読む
4. 必要に応じて地図で地域の状況を確認する（サブ）

【完了条件】
- 平常時：アプリを開いて10秒以内に判断できる
- 注意時：具体的な対策（「手洗いを徹底」など）が読める
- 毎朝プロフィール入力は不要

Story 3｜保育園からお知らせが来た
【場面】
昼12時、保育園から「今週ノロウイルスが流行しています」とLINEが来た

【ユーザーの行動】
1. アプリを開く
2. トップ画面にすでに⚠️ 注意 の診断結果が表示されている
3. Nova の診断を読む
   例）「今週の○○ちゃん（3歳・葛飾区）のリスクは中程度です。
        同じ区の3歳児クラスでノロが出始めています。
        登園前に体温と便の状態を確認し、
        異変があれば休ませてください。」
4. 地図で近隣の流行状況を確認する（サブ）
5. 登園判断・予防策を決める

【完了条件】
- 「登園させていいか」の判断材料が3分以内に得られる
- 具体的な予防アクションが提示されている
- 「このアプリは医療診断を行いません」が明示されている

Story 4｜異常検知の先行通知を受け取る（将来）
【場面】
公式データにはまだ出ていないが、Novaが異常を検知した

【ユーザーの行動】
1. プッシュ通知を受け取る
   例）「⚠️ 葛飾区で感染拡大の兆候を検知しました。
        公式データより早い段階での情報です。」
2. アプリを開いて詳細を確認する
3. 早めの対策を取る（マスク購入・手洗い強化など）

【完了条件】
- 公式発表より早くアクションできる
- 「先行シグナルであること」が明示されている

4. 非ユーザーストーリー（やらないこと）
やらないこと理由症状を入力して診断してもらう医療行為に該当するリスクがある住所（番地まで）を入力させる個人情報感が強く離脱率が上がる毎回プロフィールを入力させる毎朝の利用に摩擦が生まれる受診先の病院を紹介する医療情報の正確性担保が困難

5. UX 原則
① 平常時は10秒で閉じられる
毎朝使うアプリなので、異常がない日は極力邪魔しない。
② 不安を煽らない
「危険！」ではなく「今週は手洗いを丁寧に」という行動ベースの言葉で伝える。Nova の System Prompt でトーンを制御する。
③ 医療免責を自然に入れる
「このアプリは医療診断を行いません」を邪魔にならない形で常時表示。
④ プロフィールはデバイスに閉じる
年齢・地域はlocalStorageのみ。サーバーに個人情報を送らない設計。
